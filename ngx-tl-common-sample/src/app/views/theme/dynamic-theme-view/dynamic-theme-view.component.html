Create Theme objects
Dynamic theme service is initilized from a set of Theme objects.
Theme objects are TS structures implementing ITlTheme, providing values for each theme variables.
Example:
Set up dynamic themes of the application
Import tl-theme.service in your root app.component.ts (note: it could be done form any other root component of your application)
Register your dynamic app themes. To do that:
First create an attribute that gathers your app dynamic themes as an ITlTheme array within the app.component.ts component
Note: you can either:
  • Create your themes manually (with ITlTheme objects as defined below)
  • Use one/multiple predefined dynamic themes provided by the theme service. Prefedefined dynamic themes are designed to meet the default static themes. 3 are available for now:
    • Light Theme (add this.themeService.lightTheme within your appThemes array)
    • Dark Theme (add this.themeService.darkTheme within your appThemes array)
    • Fresh Theme (add this.themeService.freshTheme within your appThemes array)
These 3 predefined themes are the ones being registered within the frame of this app. You can try them using the tl-theme-switcher companion component
Then initialize the Theme Service calling its init() method from the ngOnInit() hook of the app.component.ts components
The following parameters shall be provided:
  • The array of all application themes you just defined
  • The default theme (to apply in case none has been stored in client local storage)
Switching theme dynamically
You can now switch theme dynamically from client side
You can do it calling the dedicated method changeTheme from the singleton ThemeService (import it from any part of the app as done before)
To change theme, only new theme name (from registered ones) is needed
Or using the dedicated theme-switcher component
