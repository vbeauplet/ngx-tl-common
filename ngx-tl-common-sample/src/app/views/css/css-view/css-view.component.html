Ngx-tl-common provides common CSS classes that can be used to:
  • Customize any HTML elements of your app keeping the ngx-tl-common consistency
  • Layout pages and containers of you application easily (create panels, tite, subtitles, deal with margin and padding)
  • ...
Pages
These classes are to use for root views of the application. Padding is automatically adapted to the chosen menu component. The following classes can be used
Class Name Description
.tl-page Basic flex-page, with horizontally centered child blocks
.tl-padded-page Flex-page, with horizontally centered child blocks. Padded, so that there is always a space around view wrt the full display
.tl-left-aligned-page Flex-page, with left-aligned child blocks
.tl-left-aligned-padded-page Flex-page, with left-aligned child blocks. Padded, so that there is always a space around view wrt the full display

Panels
Ngx-tl-common has a flex-box-oriented panel system, relying on its own implementation of basic flex items.

Two kind of flex-block can be declared:
Class Name Description
.tl-flex-block Abstract content-containers (flex panel) to contain application elements. Flex-blocks cannot be styled and is used for layouting purpose only
.tl-container-flex-block Concrete content-container (flex panel), which can be styled (background, transparency, shodwing, ...) with the tl-style system. It has a visual impact and participate to the application look
Panels (and contained child blocks) shall be sized using the tl-size system:
Name Description
.tl-full Takes the full size of the container
.tl-near-full Takes nearly the full size of the container (80%)
.tl-near-full-responsive Like before, takes nearly the full size of the container (80%). Additionnaly gets even bigger with a mobile client (90% of the container) so that no sapce around is lost
.tl-half Takes exactly half the width of the container
.tl-half-responsive Like before, takes exactly half the width of the container. However, gets nearly full-size of the container when used with a mobile client (90%) so that content can be easily read
.tl-near-half Takes nearly half the width of the container (45%)
.tl-near-half-responsive Like before, takes nearly half the width of the container. However, gets nearly full-size of the container when used with a mobile client (90%) so that content can be easily read
.tl-third Takes nearly third the width of the container (30%)
.tl-third-responsive Like before, takes nearly third the width of the container. However, gets nearly full-size of the container when used with a mobile client (90%) so that content can be easily read
.tl-quarter Takes nearly quarter the width of the container (20%)
.tl-quarter-responsive Like before, takes nearly quarter the width of the container. However, gets nearly full-size of the container when used with a mobile client (90%) so that content can be easily read
.tl-xbig Super big container (700px) when used on big screens. However responsive: on mobile, takes nearly full-container-size (90%) so that content can be easily read
.tl-big Big container (500px) when used on big screens. However responsive: on mobile, takes nearly full-container-size (90%) so that content can be easily read
.tl-medium Medium container (300px) when used on big screens. However responsive: on mobile, takes nearly full-container-size (85%) so that content can be easily read
.tl-small Small container (200px) when used on big screens. However responsive: on mobile, takes nearly full-container-size (85%) so that content can be easily read
.tl-xxsmall Very very small container (10px) for any screen size. not reponsive !
Direction of the flex-panel child elements are handled with those classes:
Class Name Description
.tl-row-dir Child blocks of the flex panel are horizontally postionned
.tl-col-dir Child blocks of the flex panel are vertically postionned
Vertical or horizontalalignment of child blocks within a flex panel do not depend on flex direction. It is handled with those classes:
Class Name Description
.tl-hor-left Child blocks of the flex panel are left aligned on the horizontal direction
.tl-hor-center Child blocks of the flex panel are centered on the horizontal direction
.tl-hor-space-between Child blocks of the flex panel are justified on the horizontal direction (space-between flex css property)
.tl-vert-center Child blocks of the flex panel are vertically centered
.tl-vert-up Child blocks of the flex panel are vertically placed from the top of the container
Concrete container flex panels can be styled using the tl-style-system classes:
Name Description
.tl-neumorphic Neumophic style, element will appear with a depth/relief effect, as if it was to come out of the screen
.tl-neumorphic-sharp Same, but the 3d effect is sharper. To use preferentially for small elements
.tl-neumorphic-outlined Same, but outlined with a thin border of soft-content-color
.tl-neumorphic-inset Same, but with the opposite 3d effect: element will apear as it is digged into the screen
.tl-neumorphic-inset-sharp Same, but the 3d effect is sharper. To use preferentially for small elements
.tl-neumorphic-dome Same, but with a "dome" effect on the element, which will appear as "convex" (rounded verticaly)
.tl-dome Element appear as "convex" (rounded verticaly)
.tl-glassmorphic Glassmorphic effect: element is transparent but element underneath appear additionnaly blurred
.tl-glassmorphic-outlined Same, but outlined with a thin border of soft-content-color
.tl-soft-transparent Element is softly-transparent, elements underneath are well-visible
.tl-sharp-transparent Same, but less transparent. elements underneath are not as well-visible
.tl-outlined Block is outlined with a thin border
.tl-shadowed Element is showed
.tl-no-style No style is applied on container
.tl-primary-style Corresponds to one of the previously described style, choice being defined by corresponding theme variable
.tl-secondary-style Corresponds to one of the previously described style, choice being defined by corresponding theme variable
Addtional advanced option classes can be used to create a panel:
Class Name Description
.tl-no-wrap Child elements or not wrapped when the main direction is "full"
Child items of flex panels can be cusomized with the following classes
Class Name Description
.tl-flex-shrink Selected child blocks can be shrinked to meet the main direction dimensions
.tl-flex-no-shrink Opposite: shrinking the child element is not permitted
.tl-flex-grow Selected child blocks grows to fill the main axis dimension
.tl-flex-full Selected child block can shrink or grow to follow the main direction dimensions
Of course both flex-panels and child blocks can be styled with generic layouting classes (margin-related, padding-related, ... see into dedicated paragraph of this page)
