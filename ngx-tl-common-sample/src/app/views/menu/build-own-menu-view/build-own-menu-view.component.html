A tl-menu-service API to build a companion components
tl-menu-service offers the following attributes to ease building up a menu component:
Name Description
size Bindable list of menu items, provided at tl-menu-service initialisation
selectedMenuItem Stores the currently selected menu item. May be refreshed automatically from route from your component using the refreshFromRoute() public method
isWrapped If needed, a flag you can set to help you store your menu wrapping status (wrapped/unwrapped, visible/hidden, ...)
unwrappedMenuItem If needed, helps you store a menu item with children that is being unwrapped (tells that sub-items shall be visible)
isInitialized Tells if service is initialized. Unless overriden by you component, it is automatically set to true after calling the init(...) method
tl-menu-service offers the following methods to affect your own menu component:
Name Description
init(menuItems: ITlMenuItem[]) See tl-menu-service initialisation
selectMenuItem(menuItem: ITlMenuItem) Natively handles selection of a menu item, shall be used (potentially overriden) from you component hen user selects a menu item. Emit the selection output of the service, potentially drives to provided route or, if none, automatically sets the menu item as unwrapped (set the unwrappedMenuItem attribute)
incrementBadgeNumber(itemId: string) Increments badge number of a menu item from its id
setBadgeNumber(itemId: string, badgeNumber: number) Set badge number of a menu item from its id
tl-menu-service offers the following events:
Name Description
selectedMenuItemSubject: Subject(ITlMenuItem) Event which is raised when a menu item is selected by user. Automatically emitted when calling the selectMenuItem method
