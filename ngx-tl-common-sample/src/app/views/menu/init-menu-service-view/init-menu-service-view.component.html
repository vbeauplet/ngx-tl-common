Why TlMenuService is a good service to handle menu within you app ?
It handles automatically selected menu item from current route

It handles wrapping status in case companion component is a wrap-capable menu

It provides a global selection event for menu items

It natively binds your menu component with ITlMenuItem objects that provide both
  • Menu Path, for routing purposes
  • Menu Label, for display purposes
  • Menu Badges, to handle notification numbers for each menu item
Create Menu Tree
Dynamic Menu service is initialized from a set of menu items (implmenting ITlMenuItem interface)
Set up menu service of the application
Import tl-theme.service in your root app.component.ts (note: it could be done form any other root component of your application)
Register your dynamic menu items themes
Display menu
Display menu shall now be handled by a companion component which uses this initialized TlMenuService

Depending on your needs, you can either:
  • Use a ngx-tl-common default menu companion component
  • Or build-up your own one which is using the TmMenuService capabilities
