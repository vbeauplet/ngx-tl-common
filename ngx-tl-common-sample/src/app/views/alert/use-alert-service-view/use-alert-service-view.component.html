Introduction
tl-alert service aims to propose simple services to:
  • Raise info, warnings or errors to user
  • Ask user for a decision or confirmation
It offers the following features:
  • Alert stack storing and initialization
  • Event emission at new alert declaration or decision taking time
  • Alert number count which can be reset at any time toshow how many alerts are read/unread
  • Automatic alert timestamping
tl-alert service shall come with a companion component (default one here) so that raised alerts/decisions are displayed on app to user
